# GitHub Pages Deployment Guide

## Overview

This project is ready to deploy to GitHub Pages using the static website files in `/site`.

The GitHub Actions workflow at `.github/workflows/deploy-pages.yml` will publish the contents of the `site` folder to GitHub Pages whenever you push to the `main` branch.

## What Gets Deployed

The published website comes from:

- `site/index.html`
- `site/episode.html`
- `site/app.js`
- `site/styles.css`
- `site/data/site-content.js`

## Recommended Repo Setup

1. Create a new GitHub repository for this project
2. Push this workspace to the `main` branch
3. In GitHub, open the repository settings
4. Go to `Pages`
5. Under `Build and deployment`, choose `GitHub Actions`
6. Push to `main` or run the workflow manually

## Suggested Repository Name

A simple option:

- `claro-english`

If you later want a custom domain, this repository name does not need to match the final website domain.

## Publish Flow

After the repo is connected:

1. Commit the current files
2. Push to `main`
3. Wait for the `Deploy GitHub Pages` workflow to complete
4. Open the Pages URL generated by GitHub

## Local Preview

You can preview locally by opening:

- `site/index.html`

or from PowerShell:

```powershell
start .\site\index.html
```

## Why This Setup Works

This project is currently a static website, so GitHub Pages is a good fit because:

- No backend is required yet
- The site content is already self-contained
- Deployment is simple and low-maintenance
- Future updates can be published with normal git pushes

## Recommended Next Step After Deployment

Once the first version is live, we should:

1. Add real brand assets into `site/assets/brand`
2. Add Episode 2 to `site/data/site-content.js`
3. Upgrade the site into a richer framework only when the content structure is stable
